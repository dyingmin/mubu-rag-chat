# mubu-rag-chat -  让幕布笔记会说话

## 1. 项目简介

你是否有很多幕布笔记，却难以快速找到需要的内容？  
**MubuAgent 让你的幕布知识库变得可对话**。

本项目针对**幕布 App 导出的文档**进行了专门优化，支持：
- 直接使用幕布导出的 Markdown/PDF/Word 文档
- 保留幕布的标题层级结构，提升检索精度
- 像聊天一样向你的幕布知识库提问
- 每个答案都能追溯到具体的幕布文档路径

> 从"记笔记"到"问笔记"，Mubu-rag-chat帮你释放幕布知识库的真正价值。

## 2. 核心功能
### 2.1 知识库构建

- 自动扫描指定文档目录
- 解析常见文档格式，例如：
  - `.txt`
  - `.md`
  - `.pdf`
  - `.docx`
- 按标题结构或内容块进行切片
- 生成并持久化向量索引
- 支持重新构建索引

### 2.2 检索与问答

- 使用 BM25 + 向量检索的混合召回策略
- 对候选上下文进行重排序
- 基于千问模型生成答案
- 回答时尽量引用命中的文档标题路径和来源文件
- 在知识不足时明确提示“知识库未覆盖”，避免编造

### 2.3 Web 交互界面

项目内置一个简洁的 Web 页面，用户可以：

- 直接输入问题并获取回答
- 一键重建索引
- 查看回答来源提示

### 2.4 评测能力

项目提供 `evaluate_rag.py` 用于批量评测 RAG 结果，支持：

- JSON / CSV 测试集读取
- 自动调用项目中的 RAG 接口生成答案
- 使用 LLM-as-judge 方式评分
- 输出单条样本结果与统计数据

## 3. 运行流程概览

项目的一般执行流程如下：

1. 加载环境变量与配置
2. 初始化目录结构
3. 扫描文档目录
4. 对文档进行解析与切片
5. 构建向量库与检索索引
6. 用户输入问题
7. 通过 BM25 + 向量检索获取候选上下文
8. 对上下文进行重排序
9. 将上下文交给大模型生成最终回答
10. 返回答案和来源信息

## 4. 依赖与外部服务

### 4.1 主要 Python 依赖

项目中可见的核心依赖包括：

- `fastapi`
- `pydantic`
- `chromadb`
- `numpy`
- `requests`
- `rank_bm25`
- `python-dotenv`
- `python-docx`（可选，用于 `.docx`）
- `pypdf`（可选，用于 `.pdf`）

### 4.2 外部服务

项目默认通过千问的 OpenAI-compatible 接口调用大模型能力，主要用于：

- 问答生成
- 评测裁判（judge）

因此需要配置：

- `QWEN_API_KEY`
- `QWEN_BASE_URL`
- `QWEN_MODEL`

## 5. 使用建议

- 将原始文档统一放入知识库目录后再构建索引
- 尽量保持文档标题层级清晰，便于分块与检索
- 评测时尽量准备标准答案和证据字段，便于量化分析
